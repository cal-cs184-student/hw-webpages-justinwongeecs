Task 5: “Pixel sampling” for texture mapping

Explain pixel sampling in your own words and describe how you implemented it to perform texture mapping.

Pixel sampling determines how texture colors are chosen when pixels are mapped in continuous UV coordinate space. UV values are not integers, so they need to be converted to discrete texel colors.

The implementation involves computing barycentric coordinates for each sample point inside the triangle and using them to interpolate the triangle’s texture coordinates. Depending on the sampling mode, these interpolated coordinates will return either a nearest-neighbor or bilinear sampled color.

Briefly discuss the two different pixel sampling methods, nearest and bilinear.

Nearest sampling chooses the closest texel whose center is closest to the UV coordinate. While it is fast to perform, it can produce more jagged results.

Bilinear sampling takes the four texels surrounding the UV coordinate and blends them based on the fractional position of UV within the cell. This method produces smoother results but requires more computation.
